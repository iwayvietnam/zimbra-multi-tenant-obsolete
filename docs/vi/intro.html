Giới thiệu về ZaaS

ZaaS là phần mở rộng tính năng cho phiên bản Zimbra Mã nguồn mở - miễn phí nhưng bị giới hạn vài tính năng. Zimbra as a Service sử dụng giao diện là web Drupal để tương tác giữa người dùng là các quản trị domain với hệ thống mail server Zimbra. Các tính năng của phần mềm:
System Admin - Quản trị hệ thống: Quản lý định mức, Quản lý tenant, Lên kế hoạch thực thi (từ trong CSDL).
Domain Admin: Quản trị hòm thư (mailbox), Quản lý alias, Quản lý group (Distributions list)
Tenant: Quản trị domain

PHPChecker

Công cụ này có vài điểm đặc biệt. Trước tiên nó dùng 1 trình duyệt ảo (HTTPClient) để gửi yêu cầu tới máy chủ Drupal để lấy lệnh. Sau đó thực thi lệnh này dưới quyền của user zimbra - trong linux, bởi vì các chương trình của zimbra (ví dụ: zmprov, zmcontrol,...) yêu cầu user thực thi phải chính xác là ZIMBRA. Vì thế user thực thi đoạn mã PHP này phải đủ quyền hạn, ví dụ như ROOT user.
HTTPClient được sử dụng trong công cụ này được lập trình bởi Simon Willison ( http://simon.incutio.com/ )

ZaaS chạy như thế nào?

Dưới đây là mô tả quá trình thực thi của ZaaS.

Trước tiên, cần cài đặt và thiết đặt trên máy chủ Zimbra (sao chép file phpchecker, đặt lịch chạy - crontab - để thực thi file php này. Bạn cần thiết đặt 1 số thứ như: địa chỉ trang Drupal, IP của server và mật khẩu cho quá trình giao dịch.
Tiếp theo, bạn cần cài đặt Drupal (nếu như bạn chưa có), và cài module : ZaaS rồi kích hoạt cho phép nó chạy. Sau đó, bạn phải tạo các role (là các kiểu quyền trong Drupal) và gán các quyền mới này cho các user ( cái mà có hậu tố tên là các domain, ví dụ: admin@zaasmhst.com. Khi đã hoàn thành, tạo các kết nối với IP, mật khẩu giống như trên máy chủ Zimbra.
Khi bạn tạo 1 lệnh cho Zimbra, bạn phải chờ 1 thời gian để máy chủ Zimbra truy cập tới Drupal và lấy lệnh thực thi đó.
Lệnh bạn tạo ra được lưu trong CSDL của Drupal. Do đã lên lịch chạy ở phía máy chủ, Zimbra sẽ định kì truy cập tới hệ thống Drupal để lấy lệnh thực thi về.
