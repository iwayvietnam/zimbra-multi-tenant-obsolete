Frequently Asked Questions

List FQA:

How does the ZaaS run?

It will explain how the ZaaS run.
First, You need install and config ( copy a phpchecker file to Zimbra server, set crontab (scheduling) to run this php file.
You need to configurate some settings: Address Zimbra, own IP, and password key for transaction.
Next, You have to install Drupal CMS (if you have not install already), Install Module: zaasmhst and enable it.
After that, create new roles and assign new permission to your users ( which have suffix name is domain name, for example : admin@zaasmhst.com. When done, Create new server connection with IP, passkey is same in your Zimbra server.
When you create a command to Zimbra service, It takes some time to get from Drupal CMS and execute it.
Your created command is saved in Database of Drupal CMS. Schedulerly, Zimbra server will connect to Zimbra server to get command.

How about other ways to execute commands?


Of cause, We have another way to execute commands.
There are 2 way to execute: run indirectly (like this version) and run directly. So what is "run directly"? It means that the command you created is executed immediately by Zimbra server.
By that way, Drupal needs to contact directly to Zimbra Server, and sends the command to it. So you nearly need not time for waiting. But what about its disadvantages?
First, If you have some mistake in commands, you can not delay or pause.

How many type (roles) Users and what can I do with those roles?


This tool uses 3 Roles to control. There are: Super Admin (highest permission), Tenant and Domain manager.
Super Admin: is user can control the system, create/stop Connections between Drupal and Zimbra server. Often this user is root account of Drupal CMS. He can assign domains management to Tenants.
Tenant: is higher permission. This user can only create Domain managers which belong to his domain(s). And he can manage all mailbox, alias, group which domain he assigned.
Domain manager: is base user. He only manage ONE domain which belong to him, no more. He can create Mailbox, Alias and Group.

What is different User Accounts between Drupal and Zimbra server?


You will see there is no relationship between Drupal and Zimbra server. You must create 2 account in same time. But according to notion, You should create Drupal user account is same user in Zimbra server (the mailbox name - email).

Must I create new Drupal user account in form like email - @company.com ?


As I said before, In notion, you should create new Drupal user account in form like Zimbra - email form such as user1@company.com It is easy to read comprehensiblely and manage. You would know who is he, where he belongs. Of cause, You can create a normal account name, none email form.
