Introdction about ZaaS

This is a Expand Features for Zimbra Open Source Version which is free but has some limit features. Zimbra as a Service uses Drupal for GUI between User (admin manager) and Service (Zimbra server). Some advanced feature such as:

System Admin: Manage quota, Manage tenant, Scheduler (retrieve/process scheduled requests from DB)
Domain Admin: Manage mailbox, Manage alias, Manage group
Tenant: Manage domains

PHPChecker

This tool uses some special things. First It uses Virtual Browser (HTTPClient) to request Drupal server to get command. And Execute this command under Zimbra permissions because zimbra execution program (for example zmprov, zmcontrol,...) require user exactly be ZIMBRA. So the user run this PHP script must be enough privilege, such as ROOT user.
HTTPClient is used in this tool is coded by Simon Willison ( http://simon.incutio.com/ )

How does the ZaaS run?

It will explain how the ZaaS run.
First, You need install and config ( copy a phpchecker file to Zimbra server, set crontab (scheduling) to run this php file.
You need to configurate some settings: Address Drupal, own IP, and password key for transaction.
Next, You have to install Drupal CMS (if you have not install already), Install Module: zaasmhst and enable it.
After that, create new roles and assign new permission to your users ( which have suffix name is domain name, for example : admin@zaasmhst.com. When done, Create new server connection with IP, passkey is same in your Zimbra server.
When you create a command to Zimbra service, It takes some time to get from Drupal CMS and execute it.
Your created command is saved in Database of Drupal CMS. Schedulerly, Zimbra server will connect to Drupal server to get command.
